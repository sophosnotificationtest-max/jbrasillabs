# Security Policy

## Supported Versions

This section lists the versions of the project that are actively supported with security updates:

| Version | Supported          |
| ------- | ----------------- |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:               |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:               |

> :information_source: Only the versions marked as supported will receive security fixes. Users on unsupported versions should upgrade to a supported version as soon as possible.

## Reporting a Vulnerability

If you discover a security vulnerability, please follow these steps:

1. **Report Privately**: Send a detailed description of the issue to our security team at **jonas.brasil@jbrasillabs.com.br**. Include:
   - Steps to reproduce the issue
   - Expected vs. actual behavior
   - Affected version(s)
   - Any relevant logs or screenshots

2. **Acknowledgment**: You will receive an acknowledgment within **2 business days** confirming that your report has been received.

3. **Investigation & Update**: Our team will investigate the issue. You will be informed of:
   - Whether the vulnerability is accepted or declined
   - Estimated timeline for a fix if accepted

4. **Resolution**: Once resolved, you will be notified when a patch or new release containing the fix is available.

> :warning: Do not publicly disclose the vulnerability until a fix has been released, to protect our users and systems.
